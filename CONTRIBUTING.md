# Contributing

Thank you for contributing to infantex-team projects.

## Before you open a PR

1. Link the related Jira ticket or spec when applicable.
2. For changes touching **auth, APIs, data, infrastructure, or dependencies**, complete the [OWASP Quick PR checklist](https://github.com/infantex-team/.github/blob/main/security-checklist.md#quick-pr-checklist).
3. Add or update tests; include at least one negative case for security-sensitive logic.
4. Do not commit secrets, credentials, or production configuration.

## Pull requests

Org repositories without their own PR template use the [default template](https://github.com/infantex-team/.github/blob/main/.github/pull_request_template.md).

## Security

See [SECURITY.md](SECURITY.md) for vulnerability reporting.

## Questions

Open a discussion or ask in your team channel if contribution expectations for a specific repo are unclear.
