# Security Policy

## Supported versions

Security fixes are provided for actively maintained production services. If you are unsure whether a version is supported, contact us before reporting.

## Reporting a vulnerability

**Do not** open a public GitHub issue for security vulnerabilities.

Report privately to:

- **Email:** security@infantex.com <!-- update to your real security contact -->
- **Subject:** `[SECURITY]` short description

Include:

1. Description of the issue and potential impact
2. Steps to reproduce (proof of concept if available)
3. Affected repository, URL, or environment
4. Your contact for follow-up

We aim to acknowledge reports within **3 business days** and provide a remediation timeline after triage.

## Safe harbor

We support good-faith security research. Do not access data that is not yours, degrade service availability, or use social engineering against staff.

## Security development standard

All infantex-team projects follow the organization [OWASP-aligned security checklist](https://github.com/infantex-team/.github/blob/main/security-checklist.md).
